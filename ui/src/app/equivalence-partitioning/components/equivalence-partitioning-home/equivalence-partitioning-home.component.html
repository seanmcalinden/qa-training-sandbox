Equivalence partitioning

Summary

Equivalance partitioning is where you divide input data into valid and invalid chunks.

Let's say we have a Username textbox with the following rules:

  • Must be at least 2 characters
  • Must not be more than 10 characters

There are 3 chunks we can create here:

Invalid Valid Invalid
0-1 2-10 11+
Whether the length of the username is 0 or 1 characters long, the validation will show the username as invalid. Where the length of the username is 2, 3, 4, 5, 6, 7, 8, 9 or 10 characters long, the validation will show the username as valid. For any username more than 10 characters long, the validation will show the username as invalid, whether the length is 11, 12, 100, 10,000 and so on.
