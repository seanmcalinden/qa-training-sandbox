Equivalence partitioning

Summary

Test coverage

When testing an input of an application (such as a textbox), we need to make sure we have enough tests to know for sure that the feature works as expected.

If we have enough tests to cover every scenario, this would mean we have 100% test coverage which is perfect.

Equivalence partitioning is a great way to work out the tests we need to gain maximum coverage.

Partitions

In equivalance partitioning, we split Valid and Invalid scenarios into groups called Partitions.

Click on the Example partitions tab to see an example.

Let's say we have a Username textbox with the following rules:

  • Must be at least 2 characters
  • Must not be more than 10 characters

There are 3 chunks we can create here:

Invalid Valid Invalid
0-1 2-10 11+
Whether the length of the username is 0 or 1 characters long, the validation will show the username as invalid. Where the length of the username is 2, 3, 4, 5, 6, 7, 8, 9 or 10 characters long, the validation will show the username as valid. For any username more than 10 characters long, the validation will show the username as invalid, whether the length is 11, 12, 100, 10,000 and so on.
